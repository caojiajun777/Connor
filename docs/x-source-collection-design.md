# X 信源采集层设计思想

本阶段目标不是实现一个自由搜索 Agent，而是构建一个稳定、可追溯、可长期运行的高质量 X 信源采集层。

## 核心定位

系统围绕人工维护的固定信源运行。信源质量由人工保证，采集层不负责判断账号是否可信，也不负责决定哪些内容值得进入日报。

采集层的唯一目标是：

> 稳定获取指定账号最近 72 小时内发布的内容，并将其整理成完整、规范、可供后续 LLM 排序的数据。

## 明确采集范围

内容必须满足：

```text
当前时间 - 72 小时 ≤ 帖子发布时间 ≤ 当前时间
```

具体约束（v0.1.1+）：

* 分页抓取，直到遇到早于 72 小时窗口的帖子，或达到安全页数上限；
* **不**因为已经凑满 N 条就停止翻页；
* 默认保留窗口内全部帖子（`max_posts_per_run: 0` = 不截断）；可选安全上限由配置/CLI 提供；
* 超过 72 小时的内容一律过滤；
* 不因为不足条目而扩大时间范围；
* 置顶帖、转发、引用、回复和原创帖均按真实发布时间判断；
* 旧的置顶帖即使展示在主页顶部，也不能进入结果。

Top 20 选择由 Milestone 2 编辑排序层完成，采集层不按时间提前裁剪“重要内容”。

## 处理顺序

```text
分页抓取账号页面内容（直到越过 72h 边界）
→ 解析帖子真实发布时间与上下文（media / quote / social_context）
→ 过滤掉 72 小时以前的帖子
→ 技术去重
→ 按发布时间倒序排列
→ 可选：按账号安全上限截断（默认不截断）
```

不能先截取页面前 10/20 条再做时间过滤，也不能在翻页时用“已够 10 条”作为停止条件。

## Coverage 语义

| 状态 | 含义 |
|---|---|
| `success` + `raw_count>0` | MCP 返回了帖子 |
| `empty_window` | 有原始帖子，但无一落入 72h |
| `fetch_returned_empty` / `mcp_empty_posts` | MCP 返回空列表（抓取失败或抖动），**不是** empty window |
| `page_incomplete` | 达到安全页数上限仍未越过窗口边界 |

`accounts_succeeded` 不包含 `raw_count=0` 的账号。

## 设计原则摘要

1. 固定信源优先于自由搜索
2. 时间范围必须明确（72h）；数量上限默认关闭，由 M2 排序
3. 原始信息尽量保留（文本、媒体元数据、引用、social_context）；语义判断后置
4. 采集层只做技术去重
5. 结果严格按真实发布时间排序
6. 单账号失败不拖垮整轮
7. 不确定性显式暴露（覆盖报告区分 fetch empty vs empty window）
8. 原始数据与清洗结果均可追溯
9. 增量状态仅在可靠成功后更新
10. 采集层保持确定性，不使用 LLM

## 与后续 LLM 层的边界

采集层输出：每个固定信源最近 72 小时内的规范化、可追溯帖子（含媒体与引用上下文字段）。

后续 LLM（M2）负责价值排序与 Top 20 精选；不负责补采缺失的 72h 内容。
